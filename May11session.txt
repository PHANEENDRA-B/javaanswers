How will you describe a wrapper class?
Every primitive data type has a wrapper class.
- It helps to perform conversion from value type to reference type aka boxing and vice versa
- short (Short) and long (Long) | int (Integer) and char (Character)
- It makes it easier to convert from String to primitive type via wrapper class and back
parseXXX() and valueOf() 
- The Java collection framework relies heavily on Wrapper class objects.

What are the naming conventions we should follow in Java?-
- A class name should be starting with capital letter
- If multiple words present then each word first letter should in ucase like ShoppingCart
- A class name should always be NOUN

- A variable name should be starting with lower case
- If multiple words present then second word onwards first letter should be in ucase like basicPay,joiningDate

- A method name should be starting with lower case
- If multiple words present then second word onwards first letter should be in ucase like editProfile(),
changePassword(), addToCart()
- A method name should always be VERB

- A package name should be entirely in lower case.
- If multiple words, the format should be separated by dot symbol like com.controller.HomeController
java.util.Date, java.util.ArrayList

We will now explore String class
We can write both the below syntax
String t="hello";... A
OR
String tt=new String("hello");... B
What is the key difference in both the styles?

... A
String object is assigned memory from a special zone of JVM heap known as String pool.
String t="hello";
String s="hello";
Two objects will NOT BE created; rather one object will be created and the memory address assigned
will be written on both the reference variables t and s. 

... B
String object is assigned from the JVM heap (not from the String pool)
Every call with new keyword allocates new memory thereby two different objects are created.

	}	
}

The String Pool (also called String Constant Pool or String Intern Pool) is a special memory 
region inside the Java heap where string literals are stored. 
Its main purpose is to optimize memory usage by reusing immutable string objects instead of 
creating duplicates.

When the JVM encounters a string literal, it:
Checks if an identical string already exists in the pool.
Reuses the existing reference if found.
Creates a new string in the pool if not found.

What do you mean by the statement "String is immutable"?
String t="hello";
t.concat(" friend");//?

String object is immutable means we cannot modify the existing content of a String object.
If tried, it creates a new object with modified content leaving the previous one unaltered.

When you create a StringBuilder instance by calling the default constructor, it creates
with initial capacity 16.
a) When the initial capacity is full, it will increase the capacity by the formula prev_cap*2+2
class Sample{	
	static public void main(String[] abc){
		StringBuilder builder=new StringBuilder();
		System.out.println(builder.capacity());//16

		builder.append("java,java,java,java,java");
		System.out.println(builder.capacity());//16*2+2=34  
	}	
}

b) When the StringBuilder instance is created using parameterized constructor, then the 
initial capacity becomes 16 plus length of the string passed as the argument.
		StringBuilder builder=new StringBuilder("java");
		System.out.println(builder.capacity());//20
