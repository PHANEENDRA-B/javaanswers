Constraints we get to use in SQL
Constraints are rules applied to the entered data into the tables.
To prevent invalid data, or duplicate data from being inserted into a column
We can add unique constraint or something equivalent.

Types
------
Primary key
Helps to identify a row from the others by a unique value.
Such a column cannot contain NULL values
Only one primary key on a table

Can I make multiple columns behave as a sole primary key?
Yes, we create a composite primary key.

Is any index created, when we add a primary key to a table?
Yes, it creates a unique index on the primary key column

UNIQUE constraint
Such a column shall not have any duplicate value

How can you define NULL in SQL terms?
A value which is unknown or unassigned.
It is not same as space or zero.

Foreign Key constraint
A column in child table that refers to primary key column of the parent table.
The parent table may be same or different from the child table.

Your manager id is the employee id of another employee

Key points
------------
Assuming the situation emp table (CHILD) and dept table (PARENT)
a) We cannot delete a dept record from the parent table if there are dependent records
present in the emp (child) table.

b) We cannot add a new record in the emp (child) table if there is no matching dept record
present in the parent table.

Exception to the rule
----------------------
ON DELETE CASCADE
If while defining the child table, we mention the clause ON DELETE CASCADE
The child records get automatically deleted if the concerned parent record is removed.

ON DELETE SET NULL
If while defining the child table, we mention the clause ON DELETE SET NULL
The respective FK column in the affected child records will be set to NULL
if the concerned parent record is removed.

How to perform any operation on a column with NULL value?

select ename,sal,comm, sal+comm as in_hand_sal
from emp
where hiredate like '2011%'

# ename	sal	comm	in_hand_sal
ALLEN	1600.00	300.00	1900.00
WARD	1250.00	500.00	1750.00
JONES	2975.00		
MARTIN	1250.00	1400.00	2650.00
CLARK	2450.00		

Cause: Any operation on a column with NULL value evaluates to NULL.
Solution: We have to modify NULL with some NOT NULL value based on data
type.

--MYSQL specific
select ename,sal,comm, sal+ifnull(comm,0) as in_hand_sal
from emp
where hiredate like '2011%';

--Oracle
vselect ename,sal,comm, sal+nvl(comm,0) as in_hand_sal
from emp
where hiredate like '2011%';

To figure out records with NULL value, use either IS NULL or IS NOT NULL.
We cannot compare two null values.

MISC function
----------------
IFNULL function converts NULL to some other NOT NULL value
NULLIF function
select nullif('ab','AB'), nullif('ab','cd')
-- if both are same returns NULL
-- if both are not same returns the first value

Another real time example of using NULLLIF to avoid arithmetic exception
--------------------------------------------------------------------------
SELECT SalesAmount / NULLIF(Quantity, 0) AS PricePerUnit
FROM Orders;
If Quantity is 0, NULLIF(Quantity, 0) returns NULL, preventing a division-by-zero error.
The result will be NULL instead of an error.

COALESCE function returns the first non null value present from a list of columns.
COALESCE can handle multiple arguments and return the first non-null value.
