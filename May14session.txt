Stream API (a trend setting addition in Java 8)

Stream will facilitate working with collections easily.
It does not hold the data rather creates a pipeline of operations.

We have two types of operations wrt stream concept
a) intermediate operation like filter, map, distinct, sorted, and so on
 -- are lazy
 -- can be chained 
b) terminal operation like count, toList, forEach , findFirst and so on
 -- only one can be used with a stream at a time

Once a stream is consumed with terminal operation, we can not use the same stream
w/o initializing it again. 

Terminal operation returns non stream return value which can be long (count()), list (toList())
or void (forEach()).
Intermediate operations are always going to return stream as output. 
Therefore, you can further pipeline the operation. 

# example of stream API

package com;

import java.util.ArrayList;
import java.util.Collections;
import java.util.List;

class Book implements Comparable<Book>{
	String title;
	float price;
	public Book(String title, float price) {
		super();
		this.title = title;
		this.price = price;
	}
	
	public String toString() {
		return String.format("Title %-15s Price %5s",title,price);
	}

	@Override
	public int compareTo(Book o) {
		// TODO Auto-generated method stub
		return Float.compare(this.price, o.price);
	}
	
}

public class Main {		
	
	public static void main(String[] args)  {

	  List<Book> books=new ArrayList<>();
	  books.add(new Book("Complete Java",312));
	  books.add(new Book("Learn Java",333));
	  books.add(new Book("Java In 24hours",276));
	  
	  for(Book bk:books) {
		  System.out.println(bk);
	  }
	 
	  //to avoid the compile time issue when sorting a 
	  //collection of custom objects we must implement
	  // the java.lang.Comparable interface and override
	  // the compareTo() method.
	  //String and all wrapper classes have already DONE so.

	  Collections.sort(books,Collections.reverseOrder());
	  System.out.println("--- after sorting ---");
	  for(Book bk:books) {
		  System.out.println(bk);
	  }	
		
	}
}
