Key differences between Statement and PreparedStatement
---------------------------------------------------------
Statement is used for simple queries without any dynamic input
"select first_name,last_name from person"

PreparedStatement is used for dynamic / parameterized queries (which take inputs at run time)

"select first_name,last_name from person where city=? and age=?"
This ? is the placeholder.

setParameter(1,"Kanpur");
setParameter(2, 21);

Statement interface is extended by PreparedStatement interface.

It pre-compiles the query and creates an optimized plan to execute some query.

ResultSet is a storage of the data fetched from the query executed from the Java application.
When first time you use the ResultSet reference to call the next() method, the pointer moves to point
to the first row in the queried data. Now, with every call, the pointer points to the next rows.
This process continues till you reach the state with no more records and get the value false.

Separation of Concerns will be very important.

The connectivity coding should be following singleton design pattern
The connection obj should be in a separate class
and then use it to obtain either Statement or PreparedStatement references.
