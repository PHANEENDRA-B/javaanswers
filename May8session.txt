When to apply HAVING clause? 
We use HAVING clause to filter the grouped data.

Can I write HAVING clause without GROUP BY clause?
 No because the fetched result will not be correct.

Does HAVING clause support column alias? Yes, if supported by the DBMS
In MySQL it does support
select deptno, count(empno) as headcount
from emp
group by deptno
having headcount >=3;

In the same query, can we use WHERE clause and HAVING clause both?
select deptno, count(empno) as headcount
from emp
where deptno >20
group by deptno
having headcount >=3

We can nest scalar and aggregate function together.

WAQ to find the length with longest name.
select max(length(ename)) from emp;

Can we nest multiple scalar functions? yes
select upper(trim(ename)) from emp;

JOINs are used to capture data from one(self join) or more tables.
Joins are of types:
a) left|right is an outer join
b) inner
c) cross join aka cartesian product
d) self join
e) equi|non equi join

Ways to write a join 
(default => inner join, only matching records from the concerned tables are fetched)

ANSI SQL style (recommended)
-----------------------------
select ename, dname
from emp join dept
on emp.deptno=dept.deptno;

Theta style (lesser readability)
---------------------------------
select ename, dname
from emp , dept
where emp.deptno=dept.deptno;

Assume I also have location table (locid, lcode, address)
which is linked to dept table based on locid column.

select ename, dname, address
from emp join dept
on emp.deptno=dept.deptno
join location
on location.locid=dept.locid;

If there are N tables to be joined, N-1 join conditions are to be written.

How to achieve the same using theta style?
select ename, dname
from emp , dept, location
where emp.deptno=dept.deptno
and location.locid=dept.locid;

We now move on to discuss sub query/ nested query in ANSI SQL.

A query when written within another query is a sub query
Can be
a) simple or non -correlated subquery :: the inner query is independent of the outer query
b) correlated sub query :: both the queries mutually depend on each other.

Can I add ORDER BY clause in the sub query?

Oracle (fails to work)
select first_name,salary,department_id
from hr.employees e where employee_id in (
	select employee_id from hr.employees where department_id=20 order by employee_id desc
)
-- order by employee_id desc;

It does not make sense becoz the outer query may return rows in any order unless
you also add ORDER BY in the outer query.

MySQL (works)
select ename,sal,deptno
from emp
where empno in (
	select empno from emp
	where deptno=20
	order by empno desc
);

Key points
a) If the inner query returns even a single NULL value, the outer query does not return any records.

We are fetching the departments in which no employees are working currently.
select deptno
from dept
where deptno not in (
	select distinct deptno
	from emp
	where deptno is not null);

--Explanation--
select deptno
from dept
where deptno not in (unique deptnos from emp table)

b) If the inner query fetches more than one result, we cannot use = or any such comparison operator.
We need to use multiple row sub query operator like IN, ANY and ALL.

select ename from emp
where job IN (
	select distinct job from emp
	where hiredate like '2008%'
);

c) Can we fetch multiple columns from the sub query and use it in the outer query? Yes

select ename,sal, deptno
from emp
where (deptno,sal) in
	(select deptno,sal from emp
	where empno=7521);

d) How shall we write a correlated sub query? The table alias is MUST in the outer query 
to reach out from the inner query

Writing an INLINE view (writing a sub query in the FROM clause)
An inline view in SQL is a subquery (SELECT statement) that appears in the FROM clause of another query.
It acts like a temporary, virtual table that exists only for the duration of the query execution.

select max(head_count)
from (
    select count(empno) head_count
    from emp
   where deptno is not null
   group by deptno
)t;

Another example of inline view (borrowed from internet!)
SELECT name, salary
FROM (
    SELECT name, salary, department,
           AVG(salary) OVER (PARTITION BY department) AS avg_salary
    FROM employees
) AS emp_avg
WHERE department = 'IT'
  AND salary > avg_salary;
How it works:

The inline view (emp_avg) calculates the average salary per department.
The outer query filters only IT employees earning above that average.

Advantages of Inline Views:
Avoids creating temporary tables.
Makes queries more readable and modular.
Can be combined with aggregation, window functions, and joins.

What is SQL view? How does it help?
A view wraps a complex query which involves aggregate functions, joins and sub query.
It does not contain any data but just the query.
It allows us to hide the complexity and enhance the security by not exposing the 
actual table name(s) and column name(s)

Can we use a view to update the records in the base table?
create view pview
as
select * from person;

We can update the base table person using simple view.
update pview
set phone='9908761234'
where name='Raman'

We cannot update the base tables through view if the view's query has complex conditions, joins and sub queries. In such as case, if it is very much required, the solution can be done using INSTEAD OF triggers.

What is the use of creating an index in SQL?
Index allows fast retrieval of data.

Any key points to remember when creating index.
a) We should create an index only on a column which is most searched and rarely updated.
b) We should not create an index on a column with low cardinality.

When we create a PK column, a unique index is automatically created on the table.
It is maintained by the database engine.
